1.为什么类里面的回调函数都是静态成员函数？

    当前这套 libevent 代码里非常典型。
    根本原因是：C 风格回调只认“普通函数指针”，不认带 this 的成员函数指针。

    1) 非静态成员函数为什么不行
    类的非静态成员函数本质签名不是你看到的 void f(...)，而是隐含了一个参数：

    真实上更接近：void f(Server* this, ...)
    所以它的函数指针类型和 C API 期望的类型不一致。
    而 libevent 这类接口通常要的是这种类型：

    void (*cb)(..., void *arg)
    这就是为什么你不能直接把 Server::read_cb（非静态）传给 bufferevent_setcb。

    2) 静态成员函数为什么可以
    静态成员函数没有 this，本质上和普通全局函数一样（只是作用域挂在类名下），所以可以匹配 C 回调签名。

2.为什么类的静态成员函数只能访问静态成员变量？

    核心原因：静态成员函数没有 this 指针。
    而普通（非静态）成员变量是“属于某个对象实例”的，访问它必须知道“是哪个对象”，也就是靠 this。

    直观理解
    静态成员函数属于类本身，调用方式常见是 ClassName::func()
    这时可能根本没有创建对象。
    非静态成员变量属于每个对象，各对象各有一份。
    比如 obj1.x 和 obj2.x 可能不同。
    所以在静态函数里直接写 x，编译器不知道你指的是哪个对象的 x。
    为什么能访问静态成员变量
    静态成员变量也是类级别，整类只有一份共享数据。
    静态函数和静态变量都不依赖具体对象，所以可以直接访问。

3.perror/fprintf+stderr/printf三种日志的使用时机
    perror 适合在刚发生系统/API失败时立刻调用，自动带上 errno 文本。
    fprintf+stderr 用于业务流程中自定义的一些错误打印
    printf（输出到 stdout）用于：正常流程信息、临时调试信息、非错误提示（如启动成功、收到请求）
